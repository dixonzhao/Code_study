3. REQUIRED FUNCTIONS
======================================================================

3.1 Source format and headers

`detect_text_format(path, max_sample_bytes=4194304) -> TextFormatInfo`

Requirements:

- open source in read-only binary/text modes;
- never rewrite the source;
- read no more than max_sample_bytes;
- try encodings in deterministic order:
  utf-8-sig, utf-8, cp932, shift_jis;
- detect comma, tab, or semicolon;
- retain a warning when csv.Sniffer and fallback disagree;
- identify the first plausible row containing CAN byte headers;
- return explicit warnings rather than silently guessing;
- support large source files without scanning data rows.

`parse_matlab_can_column(name) -> MatlabCanColumn | None`

Required example:

ID221_2_8_0

Meaning:

- CAN ID 0x221;
- Channel 2;
- DLC 8;
- Byte index 0.

CAN ID text is hexadecimal even when it contains digits only.

`discover_source_message_groups(columns) -> list[SourceMessageGroup]`

Requirements:

- group by CAN ID, Channel, and DLC;
- preserve exact original column names;
- detect duplicate byte indices;
- mark incomplete byte groups;
- do not reject incomplete groups silently.

3.2 DBC loading/parsing

`load_dbc_text(path) -> tuple[str, str]`

Return decoded text and selected encoding.

`parse_dbc(path, channel) -> list[DbcMessage]`

Support at minimum:

- `BO_` message definitions;
- `SG_` signal definitions;
- `VAL_` enum/value labels when present;
- standard and extended CAN IDs;
- Intel/little-endian and Motorola/big-endian;
- signed and unsigned;
- factor, offset, min, max, unit, receiver list.

Do not infer company meaning from signal names.
Preserve unrecognized lines by warning count in probe summary.

3.3 Decode primitives

`extract_raw_unsigned(payload, start_bit, bit_length, byte_order) -> int`

`apply_signedness(raw_unsigned, bit_length, is_signed) -> int`

`decode_physical_value(payload, signal) -> tuple[int, float]`

Requirements:

- payload accepts bytes or a list/tuple of byte integers;
- reject invalid byte values;
- reject impossible start/length combinations;
- use the verified P002 convention for Motorola bit traversal;
- return raw numeric value and physical value separately;
- physical value = raw_signed_or_unsigned * factor + offset;
- do not apply missingness, hold, or validity policy here.

3.4 Catalog

`build_signal_catalog(source_groups, dbc_by_channel) -> list[SignalCatalogRow]`

Requirements:

- match source group by Channel and normalized CAN ID;
- produce rows for matched DBC signals;
- preserve SOURCE_ONLY groups;
- preserve DBC_ONLY messages/signals in the catalog;
- mark incomplete source byte groups;
- propose ValueKind only from defensible DBC structure;
- proposed SignalRole and UpdateMode may use conservative suggestions, but
  uncertain inference must carry `inference_warning`;
- never automatically approve invalid defaults, hold, or FrameCount ownership.

3.5 Configuration serialization

`save_config_json(config, path) -> None`

`load_config_json(path) -> U001FoundationConfig`

Requirements:

- enum values serialize by stable string name;
- round trip preserves all fields;
- unknown enum names fail with an explicit error;
- no silent default replacement for an invalid file.
