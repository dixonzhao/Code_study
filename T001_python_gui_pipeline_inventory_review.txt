Python GUI Pipeline Review Exercise 01
TASK_ID: T001
STATUS: Approved for inspection only

======================================================================
1. Goal
======================================================================

Review the four existing Python GUI programs in the current company workspace
and create a factual Program Inventory.

The four program roles are:

P001 - CAN Header Catalog
P002 - MATLAB-format CAN DBC Decoder / Time-order Decoder
P003 - CAN Analysis CSV Cleaner
P004 - Stage Residual Analyzer / Excel Report Generator

This task is an inspection task only.

Do not modify source code.
Do not refactor.
Do not recommend new algorithms.
Do not create a new Analyzer.
Do not change mathematical definitions.


======================================================================
2. Working Rules
======================================================================

1. Follow the previously supplied collaboration protocol.
2. Inspect only files that the user explicitly identifies as P001-P004.
3. Base every statement on the actual current source code, GUI labels, constants,
   function definitions, and file I/O operations.
4. If information cannot be confirmed, write "UNKNOWN". Do not guess.
5. Distinguish current behavior from comments, planned behavior, and unused code.
6. Do not copy large code blocks into the result.
7. Function and class names may be listed, but function bodies must not be copied.
8. Do not upload or transmit company source code, data, DBC, screenshots, or logs.
9. Save the result only inside the company environment as:

   T001_program_inventory_audit_result.txt


======================================================================
3. Inspection Procedure
======================================================================

For each of P001-P004, inspect the program and report the fields in Section 4.

Then inspect how the four programs are used together and report:

- actual execution order;
- manual steps between programs;
- input/output filename relationships;
- overlapping responsibilities;
- missing interface definitions;
- any behavior that cannot be determined from code alone.

Pay special attention to the following operations:

- delimiter and header detection;
- CAN ID and Channel selection;
- DBC loading and signal decoding;
- endian, signedness, scale, and offset handling;
- time or Framecount sorting;
- horizontal signal expansion;
- forward fill;
- invalid-value and missing-value handling;
- column selection and dtype conversion;
- raw discrepancy calculation;
- Stage 1 and Stage 2 baseline calculation;
- final residual calculation;
- threshold and candidate flag calculation;
- persistence or consecutive-row handling;
- Event Hint window merging;
- Excel sheet generation.

Do not determine whether these methods are mathematically correct in T001.
Only determine where they exist and what the current code actually does.
