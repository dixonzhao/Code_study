======================================================================
2. FORMAL SOURCEID RULE
======================================================================

SourceId identifies the physical/logical source artifact represented by each
Update Stream row.

For the same-source Golden Replay:

- SourceId must remain exactly equal to the accepted reference Update Stream
  SourceId;
- SourceId is not a run identifier;
- SourceId must not change merely because DatasetId, RunId, output folder,
  timestamp, or execution instance changes.

Allowed replay identity changes:

- DatasetId
- RunId
- IdentityStatus
- run timestamp
- execution duration
- output folder uniqueness suffix

Not allowed for the same-source replay:

- SourceId
- row-level SourceId values
- source/input fingerprint

For a genuinely different input source in a future production run, a new
SourceId is correct and byte-equality to the old source is not expected.
